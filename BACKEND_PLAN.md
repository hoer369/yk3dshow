# 后台系统开发计划

## 技术选型
- **后端框架**: Express (Node.js) - 简单易用，生态丰富
- **数据库**: SQLite - 轻量级，无需额外配置
- **文件存储**: 本地存储 - 简单直接
- **管理界面**: 简单的HTML/CSS/JS - 无需复杂框架

## 项目结构
```
backend/
├── app.js                 # 应用入口
├── routes/
│   ├── wipers.js          # 雨刮器路由
│   ├── adapters.js        # 卡扣路由
│   └── upload.js          # 文件上传路由
├── models/
│   ├── wiper.js           # 雨刮器模型
│   ├── adapter.js         # 卡扣模型
│   └── db.js              # 数据库连接
├── public/
│   ├── admin/             # 后台管理界面
│   └── uploads/            # 上传的文件
└── package.json           # 项目配置
```

## 实现步骤
1. **初始化项目**
   - 创建Express项目
   - 安装依赖
   - 配置数据库

2. **实现API接口**
   - 雨刮器CRUD操作
   - 卡扣CRUD操作
   - 文件上传功能

3. **创建管理界面**
   - 登录页面
   - 雨刮器管理
   - 卡扣管理

4. **与前端集成**
   - 修改前端使用真实API
   - 测试完整流程

## 核心功能
- **雨刮器管理**: 添加、编辑、删除、查看
- **卡扣管理**: 添加、编辑、删除、查看
- **文件上传**: 支持3D模型(.glb)和图片上传
- **简单认证**: 基础的后台登录

## 优势
- **简单易用**: 技术栈成熟，学习成本低
- **部署简单**: 无需复杂配置
- **性能足够**: 满足小型应用需求
- **维护成本低**: 代码结构清晰，易于维护
